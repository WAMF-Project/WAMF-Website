Development Roadmap

Where WAMF is today, and where it is heading.

WAMF is under active development. This roadmap reflects the current direction of the project, but priorities may evolve through real-world testing, feedback and practical use.

Current Focus

v0.4 – Platform Stabilisation

The current release focuses on delivering a stable foundation for WAMF, improving reliability, usability and deployment for self-hosted wildlife monitoring.

  • Stable Frigate event processing
  • AI bird species identification
  • Observation storage using SQLite
  • Snapshot and clip archiving
  • Media retention tools
  • Recent detections interface
  • Activity views
  • Species metadata and reference information
  • Health and status endpoints
  • Admin tools
  • Docker deployment testing
  • Public website and documentation foundations
Next Release

v0.5 – Extensibility and Observation Management

The v0.5 release is planned to improve WAMF’s internal structure and make observations easier to explore.

  • Detection abstraction layer
  • Capture source abstraction
  • Cleaner internal provider interfaces
  • Improved API consistency
  • Observation Explorer
  • Filtering by species
  • Filtering by date range
  • Filtering by confidence
  • Improved observation browsing
  • Documentation updates for developers and users
Planned

v0.6 – Edge Devices and Distributed Capture

The v0.6 release is expected to explore edge devices and distributed capture workflows.

  • Raspberry Pi capture nodes
  • Distributed camera sources
  • Remote capture devices
  • Edge-side capture and processing experiments
  • Early support for alternative inference paths
  • Better foundations for multi-device deployments

Future Direction

Future releases may explore:

  • Additional wildlife categories beyond birds
  • More detection and classification backends
  • Improved multi-camera workflows
  • Richer statistics and activity trends
  • Better media browsing
  • Optional integrations with external dashboards
  • Community-driven plugins or extensions
  • Improved deployment options

These items are not fixed release promises. They represent areas of interest as the project grows.

Project Philosophy

WAMF is designed for wildlife, not security. Frigate provides a strong foundation for camera events and object detection. WAMF builds on that foundation with species identification, observation management, media archiving and a wildlife-focused interface.

  • Open source
  • Self hosted
  • Practical to run
  • Friendly to hobbyists and experimenters
  • Useful before it is complex
  • Extensible without becoming bloated
